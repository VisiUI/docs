### Usage Methods

1. **Externalised data**: Data such as `links`, `contact` and `socialLinks` could be provided by JSON files or APIs, facilitating integration with different platforms without changing the component code.
2. **Expandable types**: Add optional types to `socialLinks`, so as to support multiple social platforms by just adding new items to the `icon` object.
3. **Modular component**: Separate the `SocialIcons` component, the `Quick Links` block, and the `Contact` module for a more modular and maintainable structure. Each block can become a small component that can be reused.
4. **Improved accessibility**: Provide `aria-label` to social buttons for better accessibility and add functionality for keyboard navigation.
5. **Dynamic Hover**: Allow to enable or disable `hover` effects via an optional prop (e.g. `hoverEffects={false}`).
